# Pagination UI Study - Part 1 HTML 마크업

## 목표

운영자가 목록 데이터의 현재 위치와 이동 가능한 페이지를 이해할 수 있는 Pagination UI를 HTML만으로 작성한다.

이번 파트의 핵심은 페이지 번호를 나열하는 것이 아니라, 현재 페이지, 이동 가능한 페이지, 전체 데이터 규모를 명확히 전달하는 구조를 만드는 것이다.

## 구현 범위

이번 파트에서는 `index.html`을 중심으로 작성한다.

```txt
pagination-ui/study/
├─ goal.md
├─ study-summary.md
└─ index.html
```

SCSS, CSS, JavaScript는 다음 파트에서 작성한다.

## 화면 구성

```txt
페이지네이션 학습

사용자 목록 페이지 이동
운영자가 목록 데이터의 현재 위치와 이동 가능한 페이지를 확인하는 Pagination UI를 학습합니다.

총 128명 중 1-10명 표시

[이전] [1] [2] [3] [4] [5] [다음]
```

## 구현 요구사항

- 페이지의 핵심 콘텐츠는 `main` 안에 작성한다.
- 페이지 대표 제목은 `h1`으로 작성한다.
- 페이지네이션 영역은 `section`으로 묶고 `aria-labelledby`로 제목과 연결한다.
- 총 개수와 현재 표시 범위를 알려주는 텍스트를 작성한다.
- 페이지 이동 컨트롤은 `nav`로 묶는다.
- `nav`에는 `aria-label="페이지네이션"`을 작성한다.
- 이전, 다음, 페이지 번호는 `button type="button"`으로 작성한다.
- 현재 페이지 버튼에는 `aria-current="page"`를 작성한다.
- 현재 페이지가 1페이지라면 이전 버튼은 `disabled` 상태로 둔다.
- 이번 파트에서는 실제 페이지 이동 JavaScript를 작성하지 않는다.

## 데이터 조건

이번 과제에서는 아래 조건을 기준으로 마크업한다.

```txt
총 데이터 수: 128명
페이지당 표시 개수: 10명
현재 페이지: 1
전체 페이지: 13
화면에 보이는 페이지 번호: 1, 2, 3, 4, 5
현재 표시 범위: 1-10명
```

## 접근성 체크 포인트

- 페이지네이션 컨트롤은 `nav`로 묶는다.
- `nav`에는 어떤 탐색인지 설명하는 `aria-label`을 제공한다.
- 현재 페이지에는 `aria-current="page"`를 제공한다.
- 이동할 수 없는 이전/다음 버튼은 `disabled`로 표현한다.
- 페이지 번호는 단순 텍스트가 아니라 조작 가능한 `button`으로 작성한다.
- 현재 페이지를 색상이나 굵기만으로 구분하지 않는다.

## 이번 단계에서 하지 않는 것

- 페이지네이션 스타일링
- 현재 페이지 강조 스타일
- hover, focus-visible 스타일
- 이전/다음 이동 JavaScript
- 페이지 버튼 클릭 JavaScript
- 표시 범위 텍스트 동적 갱신
- React 컴포넌트화

## 완료 기준

- HTML 문법 오류 없이 작성되어 있다.
- `main`, `section`, `nav`, `button`이 역할에 맞게 사용되어 있다.
- 총 개수와 현재 표시 범위가 텍스트로 제공되어 있다.
- 이전, 페이지 번호, 다음 버튼이 있다.
- 현재 페이지 버튼에 `aria-current="page"`가 있다.
- 첫 페이지 상태에 맞게 이전 버튼이 `disabled`이다.

## 확인 방법

브라우저에서 `index.html`을 열고 다음을 확인한다.

1. 페이지 제목과 설명이 보인다.
2. 총 데이터 수와 현재 표시 범위가 보인다.
3. 이전/다음 버튼과 페이지 번호 버튼이 보인다.
4. 현재 페이지가 1페이지로 표시되어 있다.
5. DevTools Elements 탭에서 `nav aria-label`, `aria-current="page"`, `disabled`를 확인한다.

## 학습 질문

1. Pagination UI에서 현재 페이지는 왜 명확히 표시되어야 하는가?
2. 총 개수와 현재 표시 범위는 왜 필요한가?
3. 페이지 번호를 `span`이 아니라 `button`으로 작성하는 이유는 무엇인가?
4. URL 이동이 있는 페이지네이션이라면 `button` 대신 어떤 태그가 더 적절한가?
5. `aria-current="page"`는 어떤 정보를 보조기술에 전달하는가?
6. 첫 페이지에서 이전 버튼을 `disabled`로 두는 이유는 무엇인가?

## 면접에서 설명할 수 있는 문장

> Pagination UI에서는 사용자가 현재 몇 페이지에 있는지, 어느 페이지로 이동할 수 있는지, 전체 데이터 중 어느 범위를 보고 있는지 알 수 있어야 합니다.

> 현재 페이지에는 `aria-current="page"`를 제공해 시각적 강조뿐 아니라 보조기술에도 현재 위치를 전달했습니다.

> 이번 예제는 JavaScript로 현재 화면 안의 페이지 상태를 바꾸는 구조이므로 페이지 번호를 `button type="button"`으로 작성했습니다.
