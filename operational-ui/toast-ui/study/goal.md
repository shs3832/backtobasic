# Toast / Notification UI Study - Part 1 HTML 마크업

## 목표

운영 화면에서 작업 결과를 빠르게 알려주는 Toast / Notification UI를 HTML만으로 작성한다.

이번 파트의 핵심은 토스트를 예쁘게 꾸미는 것이 아니라, 성공/오류 결과를 어떤 영역에서 어떻게 알릴지, 보조기술에 어떤 강도로 전달할지 구조를 잡는 것이다.

## 구현 범위

이번 파트에서는 `index.html`을 중심으로 작성한다.

```txt
toast-ui/study/
├─ goal.md
├─ study-summary.md
└─ index.html
```

SCSS, CSS, JavaScript는 다음 파트에서 작성한다.

## 화면 구성

```txt
토스트 알림 학습

작업 결과 알림
운영자가 사용자 작업 결과를 성공 또는 오류 토스트로 확인하는 UI를 학습합니다.

[성공 토스트 보기] [오류 토스트 보기]

토스트 표시 영역
- 성공 메시지 영역
- 오류 메시지 영역
```

## 구현 요구사항

- 페이지의 핵심 콘텐츠는 `main` 안에 작성한다.
- 페이지 대표 제목은 `h1`으로 작성한다.
- 토스트 실습 영역은 `section`으로 묶고 `aria-labelledby`로 제목과 연결한다.
- 성공 토스트를 발생시키는 `button type="button"`을 작성한다.
- 오류 토스트를 발생시키는 `button type="button"`을 작성한다.
- 토스트가 표시될 컨테이너를 작성한다.
- 성공 메시지를 전달할 live region을 작성한다.
- 오류 메시지를 전달할 live region을 작성한다.
- 성공 메시지는 흐름을 크게 끊지 않도록 `aria-live="polite"`를 사용한다.
- 오류 메시지는 즉시 확인이 필요할 수 있으므로 `aria-live="assertive"`를 사용한다.

## 토스트 메시지 요구사항

```txt
성공 토스트
- 사용자 상태가 활성으로 변경되었습니다.

오류 토스트
- 사용자 상태 변경에 실패했습니다. 잠시 후 다시 시도해주세요.
```

## 접근성 체크 포인트

- 토스트를 띄우는 컨트롤은 `button`이다.
- 성공과 오류 메시지는 같은 강도로 전달하지 않는다.
- 성공 메시지는 `aria-live="polite"`로 전달한다.
- 오류 메시지는 `aria-live="assertive"`로 전달한다.
- 토스트 영역은 화면 어디에 표시될지 구조적으로 구분되어 있다.
- 닫기 버튼을 추가할 수 있는 구조를 고려한다.

## 이번 단계에서 하지 않는 것

- 토스트 위치 스타일링
- 성공/오류 색상 스타일
- 닫기 버튼 스타일
- JavaScript로 토스트 생성
- 자동 사라짐 처리
- 여러 토스트 쌓기
- React 컴포넌트화

## 완료 기준

- HTML 문법 오류 없이 작성되어 있다.
- `main`, `section`, `button`이 역할에 맞게 사용되어 있다.
- 성공/오류 토스트를 발생시키는 버튼이 있다.
- 성공/오류 live region이 분리되어 있다.
- 성공 메시지와 오류 메시지의 알림 강도가 구분되어 있다.

## 확인 방법

브라우저에서 `index.html`을 열고 다음을 확인한다.

1. 페이지 제목과 설명이 보인다.
2. 성공/오류 토스트 버튼이 보인다.
3. DevTools Elements 탭에서 성공 live region과 오류 live region을 확인한다.
4. 성공/오류 알림 강도가 다르게 작성되어 있는지 확인한다.

## 학습 질문

1. Toast UI에서 가장 먼저 전달해야 하는 정보는 무엇인가?
2. 성공 메시지와 오류 메시지를 같은 강도로 알리면 어떤 문제가 생길 수 있는가?
3. `aria-live="polite"`는 어떤 상황에 어울리는가?
4. `aria-live="assertive"`는 어떤 상황에 어울리는가?
5. 토스트는 언제 자동으로 사라지고, 언제 사용자가 직접 닫게 해야 하는가?

## 면접에서 설명할 수 있는 문장

> Toast UI는 사용자의 작업 결과를 빠르게 확인시키는 보조 피드백 UI이므로, 성공/오류 여부와 작업 대상을 명확히 전달하도록 구성했습니다.

> 성공 메시지는 흐름을 끊지 않도록 `aria-live="polite"`를 사용하고, 오류 메시지는 즉시 인지가 필요할 수 있어 더 강한 알림 영역으로 분리했습니다.
