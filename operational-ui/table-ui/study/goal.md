# Table UI Study - Part 1 HTML 마크업

## 목표

운영자가 사용자 목록을 확인하는 Admin Table UI를 HTML만으로 작성한다.

이번 파트의 핵심은 화면을 예쁘게 만드는 것이 아니라, 많은 데이터를 표 구조로 안정적으로 보여주고 `caption`, `thead`, `tbody`, `scope`를 사용해 테이블의 의미를 명확히 만드는 것이다.

## 구현 범위

이번 파트에서는 `index.html`을 중심으로 작성한다.

```txt
table-ui/study/
├─ goal.md
├─ study-summary.md
└─ index.html
```

SCSS, CSS, JavaScript는 다음 파트에서 작성한다.

## 구현 요구사항

- 페이지의 핵심 콘텐츠는 `main` 안에 작성한다.
- 페이지 대표 제목은 `h1`으로 작성한다.
- 테이블 영역은 `section`으로 묶고 `aria-labelledby`로 제목과 연결한다.
- 테이블에는 `caption`을 작성한다.
- 테이블은 `thead`와 `tbody`를 구분한다.
- 테이블 헤더 셀은 `th scope="col"`로 작성한다.
- 각 행의 대표값인 사용자 이름은 `th scope="row"`로 작성한다.
- 이메일, 역할, 상태, 가입일, 관리 액션은 `td`로 작성한다.
- 상태는 색상 없이도 의미가 전달되도록 텍스트로 작성한다.
- 관리 액션은 `button type="button"`으로 작성한다.
- 삭제 버튼은 이번 파트에 넣지 않는다.


### 가입일

```txt

태그: td

```

### 관리

```txt
역할: 행 단위 액션 제공
태그: td 안의 button
버튼: 상세, 수정
```

## 접근성 체크 포인트

- `html`의 `lang` 값은 `ko`로 작성한다.
- 테이블에 `caption`이 있는지 확인한다.
- 컬럼 제목은 `th scope="col"`로 작성한다.
- 각 행의 대표 셀은 `th scope="row"`로 작성한다.
- 상태 정보는 색상에만 의존하지 않고 텍스트로 제공한다.
- 실제 동작을 수행할 액션은 `a`가 아니라 `button`으로 작성한다.
- 버튼에는 `type="button"`을 작성해 의도하지 않은 submit 동작을 막는다.

## 완료 기준

- HTML 문법 오류 없이 작성되어 있다.
- `main`, `section`, `table`, `caption`, `thead`, `tbody`, `tr`, `th`, `td`, `button`이 역할에 맞게 사용되어 있다.
- 최소 5개의 사용자 행이 있다.
- 컬럼 헤더는 `th scope="col"`로 작성되어 있다.
- 사용자 이름 셀은 `th scope="row"`로 작성되어 있다.
- 각 행에는 `상세`, `수정` 버튼이 있다.
- CSS나 JavaScript 없이도 데이터 구조를 이해할 수 있다.

## 확인 방법

브라우저에서 `index.html`을 열고 다음을 확인한다.

1. 페이지 제목과 설명이 보인다.
2. 사용자 목록 테이블이 보인다.
3. 테이블 헤더와 본문이 구분되어 있다.
4. 각 사용자 행에 이름, 이메일, 역할, 상태, 가입일, 관리 버튼이 있다.
5. DevTools Elements 탭에서 `caption`, `thead`, `tbody`, `th scope="col"`, `th scope="row"`를 확인한다.

## 면접에서 설명할 수 있는 문장

> 운영 UI의 테이블은 많은 데이터를 반복적으로 보여주는 구조이기 때문에, 먼저 `caption`, `thead`, `tbody`, `scope`를 사용해 데이터의 의미와 관계를 명확히 잡았습니다.

> 컬럼 헤더는 `scope="col"`로, 각 행의 대표값인 사용자 이름은 `scope="row"`로 작성해 셀과 헤더의 관계를 이해하기 쉽게 만들었습니다.

> 상세나 수정처럼 현재 화면에서 동작을 수행하는 액션은 페이지 이동 목적의 링크가 아니라 버튼으로 작성했습니다.
