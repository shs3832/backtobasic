# Dropdown / Action Menu UI Study - Part 1 HTML 마크업

## 목표

운영 화면에서 자주 사용하는 사용자 작업 메뉴를 HTML만으로 작성한다.

이번 파트의 핵심은 메뉴를 예쁘게 꾸미는 것이 아니라, 메뉴를 여는 버튼과 메뉴 영역의 관계, 열림 상태, 메뉴 항목의 역할을 명확히 잡는 것이다.

## 구현 범위

이번 파트에서는 `index.html`을 중심으로 작성한다.

```txt
dropdown-ui/study/
├─ goal.md
├─ study-summary.md
└─ index.html
```

SCSS, CSS, JavaScript는 다음 파트에서 작성한다.

## 화면 구성

```txt
액션 메뉴 학습

사용자 작업 메뉴
운영자가 사용자 상태 변경과 권한 수정 액션을 실행하는 Dropdown / Action Menu UI를 학습합니다.

[상태 변경 ▼]

메뉴 열림 상태:
- 활성으로 변경
- 비활성으로 변경
- 권한 수정
```

## 구현 요구사항

- 페이지의 핵심 콘텐츠는 `main` 안에 작성한다.
- 페이지 대표 제목은 `h1`으로 작성한다.
- 액션 메뉴 영역은 `section`으로 묶고 `aria-labelledby`로 제목과 연결한다.
- 메뉴를 여는 요소는 `button type="button"`으로 작성한다.
- 메뉴 열기 버튼에는 `aria-expanded`를 작성한다.
- 메뉴 열기 버튼에는 `aria-controls`를 작성해 메뉴 영역 id와 연결한다.
- 메뉴 영역은 `id`를 가진다.
- 초기 메뉴 영역은 `hidden` 상태로 둔다.
- 메뉴 항목은 현재 화면에서 동작을 실행하는 액션이므로 `button type="button"`으로 작성한다.
- 메뉴 항목은 최소 3개를 작성한다.

## 메뉴 항목 요구사항

```txt
활성으로 변경
- 사용자 상태를 활성으로 바꾸는 액션

비활성으로 변경
- 사용자 상태를 비활성으로 바꾸는 액션

권한 수정
- 권한 수정 모달이나 패널로 확장될 수 있는 액션
```

## 접근성 체크 포인트

- 메뉴를 여는 컨트롤은 `button`이다.
- 메뉴 버튼의 `aria-expanded`는 현재 열림 상태를 표현한다.
- 메뉴 버튼의 `aria-controls`는 제어하는 메뉴 영역과 연결된다.
- 메뉴가 닫혀 있을 때 메뉴 영역은 `hidden` 상태다.
- 메뉴 항목이 페이지 이동이 아니라 현재 화면의 액션이라면 `button`을 사용한다.
- 메뉴가 열렸을 때 어떤 선택지가 있는지 HTML 구조로 확인할 수 있다.

## 이번 단계에서 하지 않는 것

- 메뉴 위치 스타일링
- 열림/닫힘 시각 스타일
- hover, focus-visible 스타일
- JavaScript 열기/닫기
- ESC 닫기
- 바깥 클릭 닫기
- 메뉴 항목 클릭 처리
- React 컴포넌트화

## 완료 기준

- HTML 문법 오류 없이 작성되어 있다.
- `main`, `section`, `button`, 메뉴 컨테이너가 역할에 맞게 사용되어 있다.
- 메뉴 열기 버튼에 `aria-expanded`, `aria-controls`가 있다.
- 메뉴 영역에 `id`와 `hidden`이 있다.
- 메뉴 항목은 `button type="button"`으로 작성되어 있다.
- 메뉴 항목이 3개 이상 있다.

## 확인 방법

브라우저에서 `index.html`을 열고 다음을 확인한다.

1. 페이지 제목과 설명이 보인다.
2. 상태 변경 버튼이 보인다.
3. DevTools Elements 탭에서 `aria-expanded`, `aria-controls`, `hidden`을 확인한다.
4. 메뉴 항목이 버튼으로 작성되어 있는지 확인한다.

## 학습 질문

1. 메뉴 열기 버튼에 `aria-expanded`가 필요한 이유는 무엇인가?
2. `aria-controls`는 어떤 관계를 표현하는가?
3. 닫힌 메뉴 영역에 `hidden`을 쓰는 이유는 무엇인가?
4. 메뉴 항목을 `a`가 아니라 `button`으로 작성하는 기준은 무엇인가?
5. Dropdown / Action Menu와 `select`는 어떤 상황에서 다르게 선택해야 하는가?

## 면접에서 설명할 수 있는 문장

> Action Menu는 현재 화면에서 상태 변경이나 권한 수정 같은 동작을 실행하는 UI이므로, 메뉴 항목을 링크가 아니라 버튼으로 구성했습니다.

> 메뉴를 여는 버튼에는 `aria-expanded`와 `aria-controls`를 제공해 현재 열림 상태와 제어 대상 메뉴의 관계를 명확히 했습니다.

> 닫힌 메뉴는 `hidden`으로 감춰 시각적으로도 보이지 않고, 보조기술에도 닫힌 상태로 전달되도록 했습니다.
